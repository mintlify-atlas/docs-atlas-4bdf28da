---
title: "Contract, VaultAsset, and Equipment type reference"
description: "Complete reference for the Contract, VaultAsset, Equipment, ChatChannel, and ChatMessage interfaces used in Zenn OS legal and asset management."
---

Zenn OS manages legal documents, digital assets, physical equipment, and team communication through four related interfaces. Contracts track the lifecycle of legal agreements from draft to execution. The Vault stores versioned digital deliverables with visibility controls that determine whether a client can see them. Equipment records track physical gear owned by the agency or by affiliate team members. Chat channels and messages power structured team communication, with channel types enforcing access boundaries between clients, affiliates, and internal staff.

## TypeScript definitions

```typescript
export type ContractStatus =
  | 'draft'
  | 'sent'
  | 'viewed'
  | 'partially_signed'
  | 'executed';

export type ContractType = 'msa' | 'nda' | 'ip_transfer' | 'contractor' | 'sow';

export interface Contract {
  id: string;
  type: ContractType;
  recipientId?: string;
  recipientName: string;
  projectId?: string;
  projectName?: string;
  status: ContractStatus;
  sentAt?: string;
  executedAt?: string;
  expiresAt?: string;
  createdAt?: string;
}

export type AssetVisibility = 'internal' | 'client_visible';

export interface VaultAsset {
  id: string;
  name: string;
  type: 'video' | 'image' | 'document' | 'audio' | 'archive' | 'design';
  size: number; // bytes
  uploadedBy: string;
  uploadedAt: string;
  milestoneId?: string;
  visibility: AssetVisibility;
  versions: { version: string; uploadedAt: string; url: string }[];
  thumbnailUrl?: string;
  isSubmittedAsProof?: boolean;
  proofVersion?: string;
  resolution?: string;
  duration?: string; // for video
  codec?: string;
}

export interface Equipment {
  id: string;
  name: string;
  type: 'camera' | 'lens' | 'lighting' | 'audio' | 'computer' | 'storage' | 'other';
  owner: 'agency' | string; // agency or userId
  status: 'available' | 'checked_out' | 'in_repair' | 'missing';
  checkedOutTo?: string; // userId
  checkedOutProject?: string; // projectId
  expectedReturn?: string;
  serialNumber?: string;
  purchaseDate?: string;
  value?: number;
  isVerified?: boolean;
  condition: 'excellent' | 'good' | 'fair' | 'needs_repair';
}

export type ChannelType = 'client_shared' | 'agency_internal' | 'executive_only' | 'direct';

export interface ChatChannel {
  id: string;
  name: string;
  type: ChannelType;
  projectId?: string;
  members: string[]; // user IDs
  lastMessageAt?: string;
  unreadCount?: number;
}

export interface ChatMessage {
  id: string;
  channelId: string;
  senderId: string;
  senderName: string;
  senderRole: UserRole;
  content: string;
  timestamp: string;
  isPinned?: boolean;
  isActionRequired?: boolean;
  attachments?: { name: string; type: string; url: string }[];
}
```

## ContractType

| Value | Description |
|---|---|
| `msa` | Master Service Agreement. Governs the overall client relationship and terms of engagement. |
| `nda` | Non-Disclosure Agreement. Sent to affiliates or clients before sharing confidential project information. |
| `ip_transfer` | Intellectual Property Transfer Deed. Executed when the agency formally transfers creative ownership to the client on project completion. |
| `contractor` | Contractor Agreement. Defines scope, deliverables, and payment terms for an affiliate working on a project. |
| `sow` | Statement of Work. A project-specific addendum detailing exact deliverables, timeline, and acceptance criteria. |

## ContractStatus lifecycle

Contracts move forward through this sequence. They do not move backward.

| Value | Description |
|---|---|
| `draft` | Document has been created but not yet sent to the recipient. |
| `sent` | Document has been dispatched to the recipient's email or portal. |
| `viewed` | Recipient has opened the document. |
| `partially_signed` | One or more signatories have signed but the contract requires additional signatures. |
| `executed` | All required parties have signed. The contract is legally binding. |

## Contract fields

<ResponseField name="id" type="string" required>
  Unique contract identifier. Format: `con_<index>` (e.g. `con_001`).
</ResponseField>

<ResponseField name="type" type="ContractType" required>
  The legal document category. See the ContractType table above.
</ResponseField>

<ResponseField name="recipientId" type="string">
  Optional `OSUser.id` of the person receiving the contract. May be absent for template drafts.
</ResponseField>

<ResponseField name="recipientName" type="string" required>
  Display name of the recipient for rendering without a user lookup.
</ResponseField>

<ResponseField name="projectId" type="string">
  Optional `Project.id` this contract is associated with.
</ResponseField>

<ResponseField name="projectName" type="string">
  Denormalised project name for display.
</ResponseField>

<ResponseField name="status" type="ContractStatus" required>
  Current state in the signing lifecycle. See ContractStatus above.
</ResponseField>

<ResponseField name="sentAt" type="string">
  ISO 8601 datetime string for when the contract was dispatched to the recipient.
</ResponseField>

<ResponseField name="executedAt" type="string">
  ISO 8601 datetime string for when all signatures were collected and the contract became executed.
</ResponseField>

<ResponseField name="expiresAt" type="string">
  Optional ISO 8601 datetime string. Applies to time-limited agreements such as NDAs.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 datetime string for when the contract record was created in the system.
</ResponseField>

## VaultAsset fields

Vault assets are versioned digital files attached to projects and milestones. Visibility controls whether a client can access the file from their portal.

<ResponseField name="id" type="string" required>
  Unique asset identifier.
</ResponseField>

<ResponseField name="name" type="string" required>
  Display filename shown in the vault browser.
</ResponseField>

<ResponseField name="type" type="'video' | 'image' | 'document' | 'audio' | 'archive' | 'design'" required>
  File category used for filtering and icon display in the vault UI.
</ResponseField>

<ResponseField name="size" type="number" required>
  File size in bytes.
</ResponseField>

<ResponseField name="uploadedBy" type="string" required>
  The `OSUser.id` of the team member who uploaded this asset.
</ResponseField>

<ResponseField name="uploadedAt" type="string" required>
  ISO 8601 datetime string for the initial upload.
</ResponseField>

<ResponseField name="milestoneId" type="string">
  Optional reference to the `Milestone.id` this asset belongs to, used when the asset is submitted as milestone proof.
</ResponseField>

<ResponseField name="visibility" type="AssetVisibility" required>
  Controls client portal access. `internal` hides the asset from the client; `client_visible` exposes it in their portal.
</ResponseField>

<ResponseField name="versions" type="{ version: string; uploadedAt: string; url: string }[]" required>
  Array of version history entries. Each entry holds the version label, upload timestamp, and a direct URL to that version of the file.
</ResponseField>

<ResponseField name="thumbnailUrl" type="string">
  Optional URL to a preview image, used for video and image assets in the vault grid view.
</ResponseField>

<ResponseField name="isSubmittedAsProof" type="boolean">
  When `true`, this asset has been formally submitted as proof of milestone completion.
</ResponseField>

<ResponseField name="proofVersion" type="string">
  The version label (from the `versions` array) that was submitted as proof.
</ResponseField>

<ResponseField name="resolution" type="string">
  Optional resolution string for image and video assets (e.g. `3840x2160`).
</ResponseField>

<ResponseField name="duration" type="string">
  Optional duration string for video and audio assets (e.g. `02:34`).
</ResponseField>

<ResponseField name="codec" type="string">
  Optional codec information for video assets (e.g. `H.264`, `ProRes 422`).
</ResponseField>

## Equipment fields

Equipment records track physical gear owned by the agency or by affiliate team members.

<ResponseField name="id" type="string" required>
  Unique equipment identifier. Format: `eq_<index>` (e.g. `eq_001`).
</ResponseField>

<ResponseField name="name" type="string" required>
  Full descriptive name of the item (e.g. `Sony FX6 Cinema Camera`).
</ResponseField>

<ResponseField name="type" type="'camera' | 'lens' | 'lighting' | 'audio' | 'computer' | 'storage' | 'other'" required>
  Equipment category used for filtering in the gear manager.
</ResponseField>

<ResponseField name="owner" type="'agency' | string" required>
  Either the literal string `agency` for agency-owned gear, or an `OSUser.id` for affiliate-owned equipment registered in the system.
</ResponseField>

<ResponseField name="status" type="'available' | 'checked_out' | 'in_repair' | 'missing'" required>
  Current physical status of the item. `checked_out` items will have `checkedOutTo` and `expectedReturn` populated.
</ResponseField>

<ResponseField name="checkedOutTo" type="string">
  The `OSUser.id` of the person currently holding the equipment. Present only when `status` is `checked_out`.
</ResponseField>

<ResponseField name="checkedOutProject" type="string">
  The `Project.id` the equipment is being used for during the current checkout.
</ResponseField>

<ResponseField name="expectedReturn" type="string">
  ISO 8601 date string for when the item is due back.
</ResponseField>

<ResponseField name="serialNumber" type="string">
  Manufacturer serial number for insurance and asset tracking purposes.
</ResponseField>

<ResponseField name="purchaseDate" type="string">
  ISO 8601 date string for when the item was acquired.
</ResponseField>

<ResponseField name="value" type="number">
  Estimated replacement or insured value in the agency's base currency.
</ResponseField>

<ResponseField name="isVerified" type="boolean">
  When `true`, the item has been physically verified by an admin during an inventory check.
</ResponseField>

<ResponseField name="condition" type="'excellent' | 'good' | 'fair' | 'needs_repair'" required>
  Physical condition rating last assessed during check-in or an inventory audit.
</ResponseField>

## ChatChannel fields

<ResponseField name="id" type="string" required>
  Unique channel identifier. Format: `ch_<index>`.
</ResponseField>

<ResponseField name="name" type="string" required>
  Slug-style channel name (e.g. `bfb-spring-campaign`).
</ResponseField>

<ResponseField name="type" type="ChannelType" required>
  Access scope. `client_shared` includes the client; `agency_internal` excludes clients; `executive_only` is restricted to admin; `direct` is a one-to-one channel.
</ResponseField>

<ResponseField name="projectId" type="string">
  Optional `Project.id` linking the channel to a specific project.
</ResponseField>

<ResponseField name="members" type="string[]" required>
  Array of `OSUser.id` values for everyone with access to the channel.
</ResponseField>

<ResponseField name="lastMessageAt" type="string">
  ISO 8601 datetime string for the most recent message, used to sort channel lists.
</ResponseField>

<ResponseField name="unreadCount" type="number">
  Number of unread messages for the current user in this channel.
</ResponseField>

## ChatMessage fields

<ResponseField name="id" type="string" required>
  Unique message identifier. Format: `msg_<index>`.
</ResponseField>

<ResponseField name="channelId" type="string" required>
  The `ChatChannel.id` this message belongs to.
</ResponseField>

<ResponseField name="senderId" type="string" required>
  The `OSUser.id` of the message author.
</ResponseField>

<ResponseField name="senderName" type="string" required>
  Display name of the sender at time of sending.
</ResponseField>

<ResponseField name="senderRole" type="UserRole" required>
  Role of the sender, used to render role badges in the chat UI.
</ResponseField>

<ResponseField name="content" type="string" required>
  The message body text.
</ResponseField>

<ResponseField name="timestamp" type="string" required>
  ISO 8601 datetime string for when the message was sent.
</ResponseField>

<ResponseField name="isPinned" type="boolean">
  When `true`, the message is pinned to the top of the channel for easy reference.
</ResponseField>

<ResponseField name="isActionRequired" type="boolean">
  When `true`, the message is flagged as requiring a response or task completion from one or more members.
</ResponseField>

<ResponseField name="attachments" type="{ name: string; type: string; url: string }[]">
  Optional array of file attachments. Each entry contains the filename, MIME type, and a direct URL.
</ResponseField>
